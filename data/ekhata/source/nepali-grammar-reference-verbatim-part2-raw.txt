खण्ड ३४: क्रियापदको विस्तृत पक्ष (ASPECT OF VERBS — DETAILED)
SECTION 34: VERB ASPECT — PERFECTIVE, IMPERFECTIVE, HABITUAL
===========================================================================

OVERVIEW:
Aspect (पक्ष) describes HOW an action unfolds — not WHEN (tense). Nepali has three
primary aspects AI must distinguish: Perfective (completed), Imperfective (ongoing),
and Habitual (repeated/customary). Chat messages often collapse aspect markers;
AI must recover intended aspect from suffix + auxiliary combinations.

34.1 PERFECTIVE (पूर्ण पक्ष) — completed/result state:
Suffix -eko/-ieko/-yeko + auxiliary (cha/xa/chhu/huncha/vayo)
Indicates action completed with result still relevant.

ma khaisakeko chhu  (I have already eaten / I am in state of having eaten)
→ ma khai sake chhu
→ ma khaisake chhu
→ ma khaisakeko xu
→ ma khaisakeko cha
→ Ma khaisakeko chhu

ma kaam sakesakeko chhu  (I have finished work)
→ ma kaam sake chhu
→ ma kaam sakesake chhu
→ ma kaam sakesakeko xu

Ram le 500 tiryo  (completed payment — perfective past)
→ Ram le 500 tireko
→ Ram 500 tiryo
→ Ramle 500 tiryo

saman becheko  (goods sold — result state)
→ maal becheko
→ saman bechieko xa
→ saman bechyo

34.2 IMPERFECTIVE (अपूर्ण पक्ष) — ongoing/continuous:
Suffix -dai/-daimai/-iraheko + auxiliary (cha/xa/chhu)
Indicates action in progress at reference time.

ma kaam gardaichhu  (I am working now)
→ ma kaam gardai chhu
→ ma kaam gardai xu
→ ma kaam gardaichu
→ ma kaam gardai cha
→ Ma kaam gardaichhu

ma khadai chhu  (I am eating)
→ ma khadai xu
→ ma khadaichhu
→ ma khaadai chhu

Ram le paisa tiraichha  (Ram is paying — ongoing)
→ Ram paisa tiraichha
→ Ram le paisa tiraichha
→ ram le paisa tiraichha

pasal khola raheko cha  (shop is opening)
→ pasal kholdai cha
→ pasal khola raheko xa

34.3 HABITUAL (अभ्यस्त पक्ष) — repeated/customary:
Simple present stem + -chhu/-chha/-chhan or past habitual -thiyo/-the
Indicates regular/repeated action or former habit.

ma harek din 6 baje uthchhu  (I wake at 6 every day)
→ ma harek din 6 baje uthchu
→ ma daily 6 baje uthchhu
→ ma harek din 6 baje uthxu

Ram harek mahina 500 tirchha  (Ram pays 500 every month)
→ Ram harek mahina 500 tircha
→ ram harek mahina 500 tirchha

pahila dherai kharcha garthe  (I used to spend a lot — past habitual)
→ pahila dherai kharcha garthe
→ pahile dherai kharcha garthe
→ pahila dherai kharcha garthiye

34.4 ASPECT + TENSE COMBINATIONS (common in chat):
Perfective + Present:  -eko chhu/xu     → ma padheko chhu (I have read)
Imperfective + Present: -dai chhu/xu    → ma padhdai chhu (I am reading)
Perfective + Past:     -eko thiyo       → ma padheko thiye (I had read)
Imperfective + Past:   -dai thiyo       → ma padhdai thiye (I was reading)

EXAMPLES (Standard → Halkhabar):
ma khaisakeko chhu
→ ma khai sake chhu
→ ma khaisake chhu
→ ma khaisakeko xu
ma kaam gardaichhu
→ ma kaam gardai chhu
→ ma kaam gardai xu
→ ma kaam gardaichu
ma padheko chhu
→ ma padheko xu
→ ma padheko cha
ma padhdai thiye
→ ma padhdai thiye
→ ma padhdai the
AI RULE: Do not confuse perfective -eko (completed) with imperfective -dai (ongoing).
-eko on financial verbs (tiryo/tireko, kinyo/kineko) = completed transaction.
-dai on financial verbs = payment/purchase in progress (rare in khata chat).

खण्ड ३५: क्रियाको वाच्य (सक्रिय, निष्क्रिय, प्रेरक)
SECTION 35: ACTIVE, PASSIVE, AND CAUSATIVE VOICE
===========================================================================

OVERVIEW:
Voice (वाच्य) indicates whether subject performs (active), receives (passive),
or causes (causative) the action. Halkhabar drops voice markers freely.

35.1 ACTIVE VOICE (सक्रिय वाच्य) — subject performs action:
[Agent] + le + [verb] — standard SOV active
Ram le 500 tiryo
→ Ram 500 tiryo
→ Ramle 500 tiryo
→ ram le 500 tiryo

maile saman kinyo
→ ma le saman kinyo
→ ma saman kinyo
→ maile maal kinyo

35.2 PASSIVE VOICE (निष्क्रिय वाच्य) — action done to subject:
[Object] + [passive marker] + [verb]
Formal: saman becheyo / saman becheko vayo
Halkhabar: saman becheko (agent omitted)

saman becheyo
→ maal becheyo
→ saman bechiyo
→ saman becheko vayo

500 tiryo (passive-like, agent dropped)
→ 500 tireko
→ 500 tirna paryo
→ paisa tireko

35.3 CAUSATIVE VOICE (प्रेरक वाच्य) — subject causes another to act:
[Causer] + le + [causee] + lai + [causative verb -aunu/-aaunu]
Ram le Shyam lai becaayo  (Ram made Shyam sell)
→ Ram le Shyam lai saman becaayo
→ Ramle Shyam lai becaayo
→ ram le Shyam lai becaayo

Ram lai tirna lagaayo  (made Ram pay)
→ Ram lai payment tirna lagaayo
→ ram lai tirna lagaayo
→ Ram lai tirna lagaayo ni

staff lai hisab garcaaye
→ karmachari lai hisab garcaaye
→ staff lai hisab garcaaye hai

35.4 VOICE IN BUSINESS CHAT:
Active (most common):  Ram le 500 kinyo / Ram 500 tiryo
Passive (rare):        500 tirna baki cha (remains to be paid)
Causative (rare):      Ram lai tirna laga / Ram lai bhannu tirnu parcha

EXAMPLES:
Ram le 500 tiryo
→ Ram 500 tiryo
→ Ramle 500 tiryo
saman becheko
→ maal becheko
→ saman bechieko xa
Ram le Shyam lai becaayo
→ Ram lai tirna lagaayo
→ staff lai hisab garcaaye
AI RULE: Default to active voice parsing. Passive/causative wrappers do not change transaction intent.

SECTION 78: FINAL AI INTERPRETATION RULES FOR NEPALI
===========================================================================

FINAL AI INTERPRETATION RULES FOR e-Khata Ollama / LLM:

RULE 1: NORMALIZATION FIRST
Always transliterate Devanagari → Roman, fold spelling aliases (Part 1 Section 31), strip emoji/particles before parsing.

RULE 2: ASPECT DISAMBIGUATION
Distinguish perfective -eko (completed: khaisakeko, tireko) from imperfective -dai (ongoing: gardaichhu, khadai) and habitual -chhu (regular: uthchhu).

RULE 3: VOICE DEFAULT
Default to active voice. Passive (saman becheko) and causative (Ram lai tirna lagaayo) do not change transaction intent.

RULE 4: AMOUNT EXTRACTION
Extract all numeric amounts; expand k/K/hajar/saya/lakh; treat '500 paisa' as Rs 500 in business context.

RULE 5: PARTY RESOLUTION
Extract party from le/lai/ko/bata patterns; use last 3 messages for ambiguous cases.

RULE 6: VERB PRIORITY
Financial verbs (kinnu, bechnu, tirnu, dinu, kharcha) override greeting/help classification.

RULE 7: DIRECTION RULES
tiryo/liyo/aayo/bata = payment IN; diyo/lai/pathayo = payment OUT or credit sale.

RULE 8: PARTICLE STRIPPING
Strip ni, ta, hai, nai, re, yaar, bhai, dai, hola, bhane before intent classification.

RULE 9: POSTPOSITION KEEP
Keep le, lai, ko, ma, bata when present; they disambiguate party and direction.

RULE 10: FUZZY MATCHING
Accept 80%+ phonetic similarity for financial verbs and voice-transcription typos.

RULE 11: CODE-SWITCH
English verbs (payment, entry, sale) combine with Nepali grammar; parse as hybrid.

RULE 12: HINDI NORMALIZE
diya→diyo, liya→liyo, kiya→garyo, kitna→kati, udhar→udhaar before parsing.

RULE 13: CONDITIONAL STRIP
Strip yadi/bhane/na bhane wrappers; extract transaction from either clause.

RULE 14: NEGATION FLIP
xaina/chaina/gardina = negate; 'tiryo xaina' = payment NOT received.

RULE 15: QUESTION DETECT
Ending ? or starting kati/ke/ko/kaha = query not entry.

RULE 16: MULTI-SPLIT
Split on 'ra', 'ani', 'and', comma for multiple transactions.

RULE 17: CONTEXT WINDOW
Include last 3 conversation turns for party/amount disambiguation.

RULE 18: HONORIFIC NORMALIZE
tirnubhayo→tiryo, garnuhuncha→garcha; honorific does not change intent.

RULE 19: TEMPORAL DEFAULT
Default transaction date to today if no hijo/voli/aaja marker.

RULE 20: ABBREVIATION EXPAND
k/K→×1000, Rs/रु→rupees, 2day→aaja, tmrw→voli before parsing.

RULE 21: TAX SEPARATION
VAT/TDS amounts may be separate; extract base transaction amount first.

RULE 22: CORRECTION DETECT
galat/sudhar/reverse/cancel/undo/delete = correction intent.

RULE 23: CONFIRMATION DETECT
thik xa/ok/done/confirm without new amount = acknowledgment only.

RULE 24: REMINDER DETECT
samjha/bhanideu/tirnu parcha = reminder, not payment received.

RULE 25: REPORT DETECT
report/ledger/balance sheet/dekhaunus = info_request not entry.

RULE 26: OUTPUT LANGUAGE
Reply in user's language style (Devanagari/Roman/English mix they used).

RULE 27: FINANCIAL SECTION BOOST
For transaction messages, boost retrieval of Sections 18, 21, 24, 26, 27, 31, 70, 79.

6. Reply in user's language register
