PART A — PRODUCT NORTH STAR
================================================================================

One sentence:
  A calm ledger desk: open app → see what needs attention + one next action →
  open a document with only the fields needed for THIS posting → advanced
  controls one click away — never gone.

What "premium + professional" means for Sutra (Busy / Tally Cloud adult software):
  - One visual language (--ds-* tokens)
  - Dense data tables, quiet chrome, sharp corners, no decorative gradients
  - Clear hierarchy: title 15px semibold → subtitle 11px muted → labels 11px → body 12px
  - Primary brand #1557b0 via tokens; never reintroduce indigo/purple/violet themes
  - Amounts: monospace, right-aligned, formatCurrency() everywhere
  - Status colors only for real status (posted/draft/cancelled/warning)

What "simplest + easiest" means:
  - Fewer things competing for the eye on first paint
  - Defaults hide rarity; power users expand (progressive disclosure)
  - One place for company / branch / FY identity (TopCommandBar)
  - Daily work in primary nav (~8–12 favourites); long tail via Command Palette
  - Same functions — relocated, collapsed, or demoted — never removed

What "full of functions" means:
  - Every existing master, voucher, report, and utility stays reachable
  - Power paths: Ctrl/Cmd+K palette, ? shortcut help, F12 config, deep links
  - Undo toasts on destructive/soft-reversible actions
  - Bookmarkable URLs (/app/:page and /app/:page/:entityId)

C3. TRANSACTION / DOCUMENT PAGE
  TransactionRouteShell or TransactionWorkspace (one document header only)
  → essential fields first
  → Optional / Rare sections collapsed by default
  → StickyActionBar for save/post
  → undo toast after post when safe

C4. DASHBOARD / HOME
  PageHeader
  → Attention strip (what needs action)
  → 3–4 KPIs max above fold
  → 4 quick actions
  → Everything else below fold or “More”

C5. DIALOG / MODAL
  Always: Dialog + DialogContent + DialogHeader + DialogTitle + DialogBody
  Never: raw position:fixed overlays for new work
  Escape + focus trap required (Radix provides)

C6. NUMBERS
  Always: formatCurrency() / formatNumber() from src/lib/utils.ts
  Table amounts: font-mono text-right (ds-financial-value)

PART E — PER-STEP WORKFLOW (HOW TO EXECUTE ANY STEP)
================================================================================

1. Identify STEP id (e.g. 1.1 LedgerMaster).
2. Open only the files named by that step (+ shared DS/routing if required).
3. Apply the matching floorplan (C1–C6).
4. Preserve all business logic, store calls, and validations.
5. Add: deep links, toast.undo, loading, formatCurrency as applicable.
6. Lint the touched files.
7. Mark the STEP [x] in THIS file.
8. Stop or continue to the next unchecked STEP — do not jump phases without
   finishing Phase 1 unless the user redirects.

Definition of Done for a migrated LIST page:
  - PageHeader with one primary action
  - EnterpriseDataTable (not raw <table> for the main list)
  - Empty + loading states
  - Create/Edit via Dialog or existing side panel
  - Delete uses toast.undo when reversible
  - Amounts use formatCurrency
  - URL deep link for entity when create/edit exists
  - No business logic regressions

Definition of Done for a migrated REPORT page:
  - ReportWorkspace chrome
  - Filters in filterSlot; KPIs in kpiSlot when summary
  - Print/export hooks wired if previously present
  - formatCurrency on money columns
  - Statutory navy .fs-report preserved where already used for print
