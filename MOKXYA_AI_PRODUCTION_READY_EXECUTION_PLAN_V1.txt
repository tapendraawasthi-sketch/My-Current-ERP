======================================================================
0. EXACT OBJECTIVE (DO NOT DRIFT)
======================================================================

Build and ship a PRODUCTION-READY, web-based, AI-embedded accounting product
for Nepal businesses where:

  1) A user can work in English, Nepali Devanagari, Romanized Nepali, and
     natural code-mix in one product surface (Ask MokXya + Accountant Mode).
  2) Shopkeeper language for a NARROW launch set becomes:
       understand → clarify if needed → draft → preview → edit →
       explicit confirm → deterministic post → real receipt → honest sync.
  3) Ask Mode answers company-scoped questions from authorized data and
     refuses / no-answers when evidence is missing (never invents current
     tax/law).
  4) Nothing posts from NL assent alone (“yes/huncha”).
  5) Calculations that hit the ledger are owned by domain engines (Dexie),
     not by the LLM or unlabeled UI totals.
  6) At least ONE launch capability matrix row reaches human
     production_approved with monitoring, rollback, and disclosed limits.

NOT the objective (reject scope creep):
  - Universal ChatGPT clone
  - Autonomous posting from model confidence
  - Guaranteed legal/tax conclusions without reviewed sources
  - Speech / OCR docs / CA workpapers / compliance calendar BEFORE Bars A–E
  - Inventing MAI-54+ feature theatre instead of closing launch gates

Success picture (done means this works in production for a real company):

  Shopkeeper types mixed Romanized/Nepali/English for a supported sale or
  purchase → correct structured preview or precise clarification → confirms
  once with a real token → sees a real receipt and honest sync badge.

  Ask Mode answers authorized company report questions, refuses when
  evidence is missing, never invents current tax/law.

  Operators can point to: one mutation story, one sync story, one production
  AI entrypoint, one released capability row with rollback.

  P1. Master Gate + step Gate satisfied with recorded evidence
  P2. Automated tests green on prod-like path (not only local toys)
  P3. Manual operator script run once on staging/prod-like company
  P4. Monitoring + rollback documented
  P5. Limitations disclosed in product copy / release note
  P6. Explicit human production_approved=true for that capability ROW
      (not for “all of AI”)
  P7. Language: EN + Devanagari + Romanized + mix proven for that row’s
      supported utterances (sample set signed)
  P8. Safety: Ask cannot mutate; NL assent cannot post; stale preview rejects;
      fake cite / tax-current-without-release abstains
